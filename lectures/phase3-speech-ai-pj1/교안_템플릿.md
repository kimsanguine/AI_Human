# Phase 3 교안 작성 템플릿

> Phase 3: 음성AI + PJ1 음성모델 (W7-W10)
> 이 문서는 Phase 3 주별 교안 작성 시 따라야 할 표준 템플릿이다.

---

## Phase 3 교안의 특징 — Phase 2와 다른 점

Phase 3부터는 **AI 채용동향 및 커리큘럼 Gap 분석**을 기반으로 실무 지향적 교안을 작성한다.

### 교안 설계 원칙 변경

| 원칙 | Phase 2 (이론 중심) | Phase 3+ (실무 중심) |
|------|-------------------|-------------------|
| **시작점** | 개념 정의 → 예제 | **문제/질문 → 솔루션 탐색** |
| **실습 비중** | 이론 60% : 실습 40% | **이론 40% : 실습 60%** |
| **실습 자료** | 교과서 데이터셋 | **실제 서비스 데이터 / PERSO AI SDK** |
| **산출물** | 주피터 노트북 | **동작하는 모델 + 문서** |
| **평가** | 퀴즈 + 과제 | **프로젝트 발표 + 코드 리뷰** |
| **Git** | 선택사항 | **필수 (모든 코드 Git 관리)** |
| **채용 연결** | 간접 언급 | **매 주차 "채용시장에서 이 스킬은" 코너** |

### Phase 3 교안에 반드시 포함할 섹션

Phase 2에는 없지만 Phase 3+에서 추가되는 필수 섹션:

1. **🎯 채용시장 연결** — 이 주차에서 다루는 스킬이 채용시장에서 어떻게 요구되는지
2. **🔧 실무 인사이트 (Ethan's Note)** — Ethan의 AI Dubbing/Avatar 실무 경험 공유
3. **📦 프로젝트 연결** — 이 주차의 학습이 PJ1에 어떻게 적용되는지
4. **✅ Git 체크포인트** — 해당 일자에 커밋해야 할 내용

## 작성 가이드라인

### 1. 질문 기반 구성 (Q-driven)

모든 이론 블록은 **질문으로 시작**한다. "Tokenizer란 무엇인가"가 아니라 **"왜 같은 문장인데 모델마다 토큰 수가 다른가?"**로 시작한다.

```
❌ 나쁜 예: "TTS란 Text-to-Speech의 약자로..."
✅ 좋은 예: "Q. AI가 사람처럼 자연스럽게 말하려면 무엇이 필요한가?"
```

### 2. Phase 2 연결점 명시

Phase 3의 모든 새 개념은 **Phase 2에서 배운 것의 확장**으로 설명해야 한다.

```
✅ "Phase 2에서 배운 Attention Mechanism이 TTS의 정렬(alignment)에서 핵심 역할을 한다"
✅ "Phase 2의 RNN이 음성 시퀀스 모델링에 왜 적합한지 복습하자"
```

### 3. 실습 비중 60% 이상

| 시간대 | 배분 |
|--------|------|
| 오전 (3h) | 이론 1.5h + 개념 확인 실습 1.5h |
| 오후 (4h) | 심화 실습 3h + 정리/Q&A 1h |
| **합계** | 이론 1.5h (21%) + 실습 4.5h (64%) + 정리 1h (14%) |

### 4. Ethan's Note — 실무 인사이트

매일 최소 1회, **Ethan의 실무 경험**을 공유하는 코너를 포함한다.

포맷:
```markdown
🔧 **Ethan's Note**: {제목}
> {실무에서 겪은 구체적 사례 — 2-3문장}
> {수강생에게 주는 시사점}
```

주제 예시:
- AI Dubbing 서비스에서 TTS 품질을 올리기 위해 시도한 것들
- 실서비스에서 Whisper STT의 한국어 성능 이슈와 해결법
- PM 관점에서 음성 모델 선택 시 고려한 트레이드오프
- "모델 성능이 0.5% 올랐는데 출시할 수 있나?" — 의사결정 기준

### 5. Git 체크포인트

Phase 3부터 **모든 실습 코드는 Git으로 관리**한다. 매일 교안 끝에 Git 커밋 포인트를 명시한다.

```markdown
✅ **Git 체크포인트**:
- `git add . && git commit -m "W7D1: TTS 기초 실습 - Tacotron2 분석"`
- `.gitignore`에 모델 체크포인트(*.pt, *.bin) 추가 확인
```

### 6. 프로젝트 연결

W7(이론 주차)에서도 **"이 내용이 PJ1에서 어떻게 쓰이는지"**를 언급하여, PJ1 시작 전에 수강생이 프로젝트 전체 그림을 머릿속에 그릴 수 있게 한다.

```markdown
📦 **프로젝트 연결**:
> 오늘 실습한 VITS TTS는 PJ1에서 "나만의 음성 모델"의 베이스라인이 된다.
> PJ1 D3에서 이 모델을 PERSO AI SDK에 탑재하는 작업을 하게 된다.
```
