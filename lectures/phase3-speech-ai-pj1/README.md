## Phase 3가 특별한 이유

Phase 3는 **"이론에서 프로덕트로"** 전환이 일어나는 첫 번째 Phase다.

Phase 1-2에서 Python, ML/DL, NLP 기초를 다졌다면, Phase 3에서는 **실제 PERSO AI 제품에 탑재되는 음성 모델을 직접 만든다.** 수강생이 처음으로 "내가 만든 모델이 서비스에 들어갈 수 있다"는 경험을 하는 시점이다.

### Phase 2와의 핵심 차이

| 구분 | Phase 2 (ML/DL + NLP) | Phase 3 (음성AI + PJ1) |
|------|----------------------|----------------------|
| 목적 | AI 기초 체력 확보 | **실무 모델 개발 + 첫 프로젝트** |
| 학습 방식 | 이론 → 실습 (교과서적) | **문제 → 솔루션 (프로젝트 기반)** |
| 산출물 | 개인 실습 노트북 | **PERSO AI SDK 탑재 가능한 음성 모델** |
| 채용시장 연결 | 간접적 (기초 역량) | **직접적 (Gap 분석 기반 실무 스킬)** |
| 평가 | 퀴즈 + 과제 | **프로젝트 발표 + 동료 리뷰** |

### 채용동향 Gap 분석 반영

> 출처: [AI 채용동향 및 커리큘럼 Gap 분석] — 4대 Gap 해소 전략

| Gap | 반영 내용 | 적용 시점 |
|-----|---------|---------|
| ③ Git/GitHub | PJ1에서 Git 기반 프로젝트 관리 실전 적용 | W8-W10 전체 |
| (예비) ① LoRA | W10에서 LLM 기초 입문 시 "Fine-tuning이란?" 개념 소개 → Phase 4 LoRA 연결 | W10 D3-D5 |

**채용시장에서 요구하는 음성AI 역량:**
- STT/TTS 모델 이해 및 활용 (Whisper, VITS 등)
- 음성 데이터 전처리 + 평가 파이프라인
- 오픈소스 모델 커스터마이징 경험
- **"프로젝트에서 모델을 직접 만들어봤다"는 경험**

## 주차별 개요

### W7 (Day 31-35): TTS/STT 모델 개발

**핵심 질문:** "기계는 어떻게 말하고, 어떻게 듣는가?"

| 일차 | 주제 | 핵심 실습 |
|------|------|---------|
| D1 | TTS 이론: 음성 합성의 원리 | Tacotron2 아키텍처 분석 |
| D2 | TTS 실습: VITS + 한국어 TTS | VITS로 한국어 음성 합성 |
| D3 | STT 이론: 음성 인식의 원리 | CTC, Attention-based 디코딩 |
| D4 | STT 실습: Whisper 심화 | Whisper 모델 크기별 비교 실험 |
| D5 | TTS+STT 통합 파이프라인 | 음성→텍스트→음성 라운드트립 |

**RD 매핑:** 과목5 "음성 데이터 활용 TTS와 STT 모델 개발" — 40h (Phase 3 배분 48h 중)

### W8 (Day 36-40): STT 완성 + PJ1 시작

**핵심 질문:** "내가 만든 음성 모델이 PERSO AI에 어떻게 들어가는가?"

| 일차 | 주제 | 핵심 실습 |
|------|------|---------|
| D1 | STT 고급 (화자 인식, 감정 분석) + 모델 평가 종합 | 멀티스피커 STT + 평가 메트릭 |
| D2 | PJ1 오리엔테이션 + 기획 (요구사항 분석) | PERSO AI SDK 구조 파악 + 기획서 |
| D3 | PJ1: 아키텍처 설계 + 데이터 수집/전처리 | 설계 문서 + 데이터 파이프라인 |
| D4 | PJ1: 모델 구현 시작 (베이스라인) | 베이스라인 모델 학습 |
| D5 | PJ1: 모델 학습/실험 + W8 스프린트 리뷰 | 실험 결과 비교 + 회고 |

### W10 (Day 46-50): PJ1 완료 + LLM 기초 입문

**핵심 질문:** "음성 모델 다음은? LLM이 어떻게 음성과 연결되는가?"

| 일차 | 주제 | 핵심 실습 |
|------|------|---------|
| D1 | PJ1 최종 점검 + 발표 준비 | 발표자료 작성 + 리허설 |
| D2 | **PJ1 발표** + 동료 리뷰 + Phase 3 전반 회고 | 프로젝트 발표 |

| D5 | STT→LLM→TTS 연결 + Phase 4 프리뷰 | 음성 어시스턴트 프로토타입 |

**RD 매핑:** PJ1 완료 (8h) + 과목6 "거대 언어 모델 자연어 생성" 시작 (24h)

## 3-Layer 시스템 적용

| Layer | Phase 3 적용 |
|-------|-------------|
| **L1: AI Pair Coding** | ChatGPT/Claude로 음성 처리 코드 디버깅, 모델 파라미터 탐색 |
| **L2: Product Thinking** | "이 음성 모델이 PERSO AI 사용자의 어떤 문제를 푸는가?" |
| **L3: Thought Process** | PJ1 기획서, 설계 문서, 회고록을 통한 사고 과정 기록 |

### Phase 2 → Phase 3 L-Layer 진화

| Layer | Phase 2 | Phase 3 |
|-------|---------|---------|
| L1 | AI로 코드 작성 보조 | AI로 **모델 실험 설계** + 코드 생성 |
| L2 | "이 기술이 왜 중요한가?" | **"이 모델이 제품에 어떻게 쓰이는가?"** |
| L3 | 개인 학습 노트 | **프로젝트 문서 (기획→설계→구현→회고)** |

## Ethan's Teaching Notes

### 음성AI 강의 시 활용할 실무 경험

Ethan은 AI Dubbing 제품을 리딩한 경험이 있다. Phase 3에서 공유할 실무 인사이트:

- **AI Dubbing에서의 TTS 품질 이슈**: 자연스러운 운율(prosody), 감정 표현, 다국어 대응
- **실서비스에서의 STT 한계**: 노이즈, 방언, 전문용어 처리
- **모델 선택 의사결정**: 오픈소스(Whisper) vs 상용 API(Google STT) 트레이드오프
- **Product Manager 관점의 음성AI**: 사용자 피드백 → 모델 개선 사이클

### 주의사항

- W8부터 PJ1이 시작되므로, **W7에서 TTS/STT 실습을 확실히 마무리**해야 함
- GPU가 필요한 실습(VITS 학습 등)은 **Colab 환경 사전 준비 필수**
- PJ1은 **개인 프로젝트** (팀 프로젝트는 PJ3에서 시작)
- W10 후반의 LLM 입문은 **맛보기 수준** — Phase 4에서 본격 진행
