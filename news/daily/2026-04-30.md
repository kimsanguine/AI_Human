### 1. PSP — Indic TTS의 "악센트"를 6개 음운 축으로 분해하는 첫 해석가능 벤치마크 (arXiv 2604.25476)
**출처:** [arXiv — 2026-04-28](https://arxiv.org/abs/2604.25476)

WER·CER·MOS 같은 기존 TTS 평가는 "발음은 맞는데 원어민 같지 않다"는 잔여 악센트 차이를 잡지 못한다는 문제의식에서 출발한 연구다. PSP(Phoneme Substitution Profile)는 권설음 붕괴율(RR), 기식 충실도(AF), 모음 길이 충실도(LF), 타밀 zha 충실도(ZF), 음운 임베딩 공간의 Fréchet Audio Distance(FAD), 운율 신호 발산도(PSD)의 6개 차원으로 악센트를 분해한다. 평가는 합성 음성을 Wav2Vec2-XLS-R char-CTC 정렬기로 강제 정렬한 뒤 각 음소 구간 임베딩을 원어민 프로토타입과 코사인 유사도로 비교하며, ElevenLabs v3·Cartesia Sonic-3·Sarvam Bulbul을 힌디·텔루구·타밀로 교차 평가했다.

> **학습 연결:** Ch04 음성 데이터 처리에서 배우는 "음소 단위 정렬(forced alignment)"과 "임베딩 기반 유사도"가 다국어 TTS 품질 평가의 표준 평가 대신 어떻게 음운 단위까지 분해되는지 보여주는 사례다. WER 한 숫자로 보이지 않던 음소별 손실을 임베딩 코사인 유사도로 가시화한다는 점에서, 한국어 TTS의 "어색함"도 같은 방식으로 해부할 수 있다.

### 2. In-Sync — Speech-Aware LLM에 단어 단위 타임스탬프 예측을 결합한 ASR (arXiv 2604.22817)
**출처:** [arXiv — 2026-04-26 무렵](https://arxiv.org/abs/2604.22817)

In-Sync는 사전학습된 음향 모델과 LLM을 결합하는 SpeechLLM(AudioPaLM, SpeechGPT, SALMONN, SLAM-LLM 계열)의 약점인 "단어 단위 시간 정보 부재"를 직접 겨냥한다. 제안 모델은 음향 인코더 출력에 LLM 디코더가 단어별 시작·종료 타임스탬프를 함께 생성하도록 추가 학습을 수행해, ASR 정확도와 시간 정합성을 동시에 확보한다. 자막 생성·다이어라이제이션·동기 더빙처럼 "언제"가 중요한 응용에 그대로 결합 가능한 구조다.

> **학습 연결:** Ch04에서 다루는 "어텐션 메커니즘(attention mechanism)"과 "토큰 단위 정렬"이 텍스트→음향 매핑으로 확장되는 지점이다. LLM 디코더가 텍스트뿐 아니라 시간(timestamp)까지 다중 출력으로 예측한다는 설계는, Ch06에서 배울 LLM 토큰 디코딩 전략이 음성 데이터에서 어떻게 시계열 구조와 결합하는지를 구체적으로 보여준다.

### 3. Praxy Voice — 비-Indic 베이스에서 상업급 Indic TTS를 만드는 프롬프트 회복 + BUPS 기법 (arXiv 2604.25441)
**출처:** [arXiv — 2026-04-28](https://arxiv.org/abs/2604.25441)

Praxy Voice는 ElevenLabs·Cartesia 같은 영어 중심 베이스 모델을 동결(frozen)한 채 Indic 언어로 옮기되, 상업적 학습 데이터 비용을 0으로 만드는 두 가지 기법을 결합한다. (1) Voice-Prompt Recovery는 합성 결과의 운율·음색 표류를 짧은 프롬프트 임베딩으로 보정하고, (2) BUPS(Backbone-Unaware Prompt Steering)는 베이스 모델 내부에 접근하지 않고도 Indic 음운 분포로 출력을 끌어당긴다. 결과적으로 가중치 재학습 없이 힌디·타밀에서 상업급 자연성을 확보했다고 보고한다.

> **학습 연결:** Ch04의 "전이 학습(transfer learning)"과 "프롬프트 임베딩"이 음성 도메인에서 어떻게 비용 0의 다국어 확장으로 이어지는지 보여준다. 한국어처럼 학습 데이터가 부족한 언어 환경에서, 영어 베이스 모델을 동결한 채 임베딩 공간만 조정하는 전략이 실제로 통한다는 증거다 — Ch07 프롬프트 엔지니어링에서 다룰 "임베딩 조작" 개념의 음성 버전이라 볼 수 있다.

### 4. Anthropic, $900B+ 가치로 추가 1차 라운드 검토 — OpenAI 추월 가시권 (4월 29일)
**출처:** [Bloomberg — 2026-04-29](https://www.bloomberg.com/news/articles/2026-04-29/anthropic-considering-funding-offers-at-over-900-billion-value)

Anthropic이 9,000억 달러 이상 기업가치로 새 1차 펀딩을 검토 중이다. 24일 발표된 Google의 최대 400억 달러 투자(현금 100억 + 성과조건부 300억)에 이어 4월 24일 세컨더리에서 1조 달러를 찍은 시장 평가가 이번엔 1차 라운드로 옮겨붙는 흐름이다. 회사 측은 "초기 검토 단계, 어떤 제안도 수락하지 않았다"는 입장이다. 성사되면 OpenAI(약 8,800억) 대비 상위로 올라서며, 이는 4월 28일 OpenAI·Anthropic이 미 하원 국토안보위원회에 사이버 모델을 별도 브리핑한 직후의 자본 시그널이다.

> **학습 연결:** AI 산업의 자본 흐름이 "펀더멘털 모델 → 사이버·국방 안보 → 자본 재집중"의 순환에 들어섰다는 신호. Ch04 NLP 모델이 단순 텍스트 처리가 아니라 사이버 보안 코드 분석까지 확장되며 정책·자본 의사결정의 변수가 된다는 점을 보여준다.

### 5. Amazon, AWS Bedrock에 OpenAI 모델·Codex·Bedrock Managed Agents 추가 (4월 28일)
**출처:** [TechCrunch — 2026-04-28](https://techcrunch.com/2026/04/28/amazon-is-already-offering-new-openai-products-on-aws/)

AWS Bedrock이 OpenAI 최신 모델, 코드 생성 서비스 Codex, 그리고 OpenAI 추론 모델 기반의 신규 에이전트 빌더 "Bedrock Managed Agents"를 동시에 탑재했다. Managed Agents는 에이전트 행동 조향(steering)과 보안 정책을 기본 제공하며, 같은 날 Amazon은 상품 페이지에서 음성 Q&A로 답을 생성하는 실시간 음성 응답 기능도 공개했다. 4월 24일 Anthropic-AWS 1,000억 달러 10년 약정과 별개로, AWS가 모델 공급사를 멀티 벤더로 가져가는 전략이 명확해지는 흐름이다.

> **학습 연결:** Ch08 Langchain·MCP 단계에서 다룰 "Tool Use 에이전트"가 클라우드 PaaS의 기본 기능으로 내려앉는 사례다. 모델 자체보다 "에이전트 조향(steering)·관측(observability)·보안" 레이어가 차별점이 되는 지점이며, B2B SaaS 입장에서는 Bedrock Managed Agents 같은 매니지드 레이어 위에 도메인 로직만 얹는 방향이 합리적 선택이 된다.
