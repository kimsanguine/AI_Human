### 1. DriftSE — "표류 모델(Drifting Models)" 기반 1-step 음성 향상 (Interspeech 2026 제출)
**출처:** [arXiv:2604.24199 — 2026-04-27](https://arxiv.org/abs/2604.24199)

음성 노이즈 제거(speech enhancement)를 균형 문제(equilibrium problem)로 새롭게 정식화한 생성 프레임워크다. 기존 diffusion 기반 음성 향상이 수십 step의 반복 추론을 요구하는 데 반해, DriftSE는 mapping function의 pushforward 분포를 깨끗한 음성 분포에 직접 정렬해 **단 1-step 추론**으로 동등 이상의 품질을 낸다.

> **학습 연결:** Ch04에서 학습하는 음성 데이터의 **시간-주파수 표상(spectrogram)**과 **확률 분포 매핑** 개념이, 어떻게 generative 음성 모델의 추론 비용을 줄이는 실전 기법으로 이어지는지 보여주는 사례다. 토크나이저가 텍스트를 표상하듯, 음성도 결국 분포 정렬 문제로 환원된다.

### 2. Korean-Centric LLM Token Pruning — 다국어 LLM을 한국어 NLP에 최적화하는 어휘 압축 벤치마크
**출처:** [arXiv:2604.16235 — 2026-04-22](https://arxiv.org/abs/2604.16235)

Qwen3·Gemma-3·Llama-3·Aya를 대상으로 **token pruning(불필요 언어 토큰·임베딩 파라미터 제거)**을 적용하여 Original / EnKo(영-한) / EnKoZh(영-한-중) 세 어휘 구성에서 한국어 NLP 성능을 비교한다. 결론은 명확하다 — 토큰 pruning은 **언어 혼동(language confusion)**을 줄여 생성 안정성을 높이고, 한국어 기계번역에서는 오히려 성능이 향상된다.

> **학습 연결:** Ch04의 **서브워드 토크나이저(BPE/SentencePiece)**·**임베딩 테이블** 개념의 직접 응용이다. 토크나이저는 단순히 자르는 도구가 아니라 **모델 용량을 어디에 배분할지 결정하는 정책 레이어**임을 보여주며, 한국어 같은 비라틴 언어가 다국어 LLM에서 손해를 보는 구조적 이유를 토큰 단위로 진단한다.

### 3. Do LLM Decoders Listen Fairly? — ASR에서 LLM prior가 만드는 편향 벤치마크
**출처:** [arXiv:eess.AS — 2026-04-27](https://arxiv.org/list/eess.AS/recent)

Speech-aware LLM 기반 ASR이 빠르게 표준이 되는 가운데, **LLM의 언어 prior가 인식 결과에 어떤 편향(성별·방언·코드스위칭)을 주입하는지** 정량화한 벤치마크다. 같은 시점 발표된 "In-Sync"는 ASR을 위한 음성 인식 LLM 어댑테이션에 **단어 단위 timestamp 예측**을 결합해 자막·검색 사용성을 끌어올린다.

> **학습 연결:** Ch04의 **어텐션 메커니즘**과 **언어 모델링 확률(P(token|context))** 개념이 음성으로 확장될 때 무엇이 깨지는지 보여준다. ASR이 더 이상 음향 모델 단독이 아니라 **음향 + 디코더 LLM의 합산 분포**로 동작한다는 사실은 평가 지표(WER) 너머 **공정성 평가**가 필수임을 시사한다.

### 4. Google, Pentagon에 AI 클래시파이드 네트워크 접근 확대 — Anthropic이 거부한 자리에 진입
**출처:** [TechCrunch — 2026-04-28](https://techcrunch.com/2026/04/28/google-expands-pentagons-access-to-its-ai-after-anthropics-refusal/)

미 국방부(DoD)가 클래시파이드 네트워크에서 모든 합법적 용도로 AI를 사용할 수 있도록 Google이 접근 범위를 확대했다. 앞서 Anthropic은 **국내 대량감시·자율무기 사용 차단**을 가드레일로 요구하다 거부했고, 그 자리를 OpenAI·xAI에 이어 Google이 이어받았다. Google 계약에는 동일 취지의 문구가 포함됐지만 **법적 강제력 여부는 불분명**하다는 평가다.

> **학습 연결:** Ch04~05에서 다루는 음성·언어 모델이 **민감 도메인(국방·의료·금융)**에 배치될 때, 모델 자체의 정확도보다 **사용 정책·가드레일 설계**가 채택을 결정하는 변수로 부상한다. CPO 관점에서는 "정책을 코드로 푼 회사"가 시장을 가져간다는 신호다.
