### 1. xAI, Grok STT·TTS API 단독 출시 — 전화 통화 인식 에러율 5%로 업계 최저
**출처:** [MarkTechPost — 2026-04-18](https://www.marktechpost.com/2026/04/18/xai-launches-standalone-grok-speech-to-text-and-text-to-speech-apis-targeting-enterprise-voice-developers/)

xAI가 Grok 모바일·테슬라·Starlink 고객지원에서 이미 쓰이던 자체 음성 스택을 개발자 API로 공개했다. STT는 25개 언어·화자 분리·워드 타임스탬프·Inverse Text Normalization을 지원하고 배치 $0.10/시간·스트리밍 $0.20/시간으로 가격을 공격적으로 매겼다. 핵심 지표로 전화 통화 엔티티 인식 에러율 5.0%를 내세웠는데, 이는 ElevenLabs 12.0%, Deepgram 13.5%, AssemblyAI 21.3% 대비 절반 이하의 수치다.

> **학습 연결:** Ch04 후반에서 배우는 **음성 특징 추출(MFCC)**과 **CTC 기반 시퀀스 정렬**이 상용 API에서 어떻게 구현되는지 보여주는 사례다. 특히 "Inverse Text Normalization"은 토크나이저가 "1 2 3" → "123"처럼 출력 문자열을 후처리하는 과정으로, Ch04 토크나이저 파트와 직결된다.

### 2. Google Gemini 3.1 Flash TTS 공개 — 200+ 오디오 태그로 "연출 가능한" TTS 시대 개막
**출처:** [MarkTechPost — 2026-04-15](https://www.marktechpost.com/2026/04/15/google-ai-launches-gemini-3-1-flash-tts-a-new-benchmark-in-expressive-and-controllable-ai-voice/)

Google이 프리뷰 공개한 Gemini 3.1 Flash TTS는 200개 이상의 오디오 태그를 입력 텍스트에 임베딩하여 톤·템포·억양·감정을 문장 단위로 제어한다. 70+ 언어, 다화자 대화를 네이티브 지원하며 Artificial Analysis TTS 리더보드에서 Elo 1,211점으로 ElevenLabs v3를 제치고 2위에 올랐다. 모든 출력 오디오에는 Google의 SynthID 워터마크가 삽입된다.

> **학습 연결:** Ch04 임베딩 파트에서 배우는 **입력 토큰 임베딩의 조건부 제어(conditional embedding)**가 실제 TTS 파이프라인에서는 "오디오 태그"라는 형태로 구현된다. 태그 토큰 하나하나가 **style embedding vector**로 변환되어 디코더의 어텐션에 주입되는 구조다.

### 3. arXiv 신작 — Token Pruning으로 한국어 중심 LLM 최적화 (Qwen3·Gemma-3·Llama-3·Aya 벤치마크)
**출처:** [arXiv — 2026-04-17](https://arxiv.org/abs/2604.16235)

"Optimizing Korean-Centric LLMs via Token Pruning" 논문은 다국어 LLM의 어휘·임베딩 파라미터 중 한국어 서비스에 불필요한 토큰을 제거하는 압축 기법을 Qwen3, Gemma-3, Llama-3, Aya 네 모델에 적용해 체계적으로 벤치마크했다. Original / EnKo(영한) / EnKoZh(영한중) 세 가지 어휘 구성을 비교한 결과, 토큰 프루닝이 생성 안정성을 크게 개선하고(언어 혼동 제거) 한국어 기계번역 성능을 자주 향상시켰다. 메모리 제약이 있는 도메인 특화 배포에 특히 효과적이다.

> **학습 연결:** Ch04 토크나이저 파트에서 배우는 **BPE(Byte-Pair Encoding) 어휘 구성**과 **서브워드 토크나이제이션**이 왜 중요한지 보여주는 실무 사례다. 한국어 전용 서비스에서 중국어·아랍어·힌디어 토큰을 보유하면 "multilinguality의 저주"로 임베딩 레이어 파라미터가 낭비되는데, 이 논문은 어휘 크기 축소가 오히려 한국어 성능을 높일 수 있음을 증명한다.

### 4. Google, 신세대 추론 전용 TPU로 Nvidia 정면 도전 — Meta·Anthropic·Apple도 TPU 채택
**출처:** [Bloomberg — 2026-04-20](https://www.bloomberg.com/news/features/2026-04-20/google-eyes-new-chips-to-speed-up-ai-results-challenging-nvidia)

Google이 이번 주 신형 TPU(Tensor Processing Unit) 세대를 공개할 예정이라고 Bloomberg가 보도했다. 이번 TPU는 학습이 아닌 **추론(inference)**에 특화된 것이 핵심이다. 현재 Meta, Anthropic, Apple이 모두 TPU를 쓰고 있어 Nvidia GPU가 장악한 AI 인프라 시장의 견고함이 처음으로 흔들리는 모습이다. Marvell이 Google TPU + 메모리 처리 유닛 제작을 돕는다는 루머로 주가가 급등했고, Broadcom 주가는 하락했다.

> **학습 연결:** 학습(training)과 추론(inference)의 하드웨어 요구사항이 다르다는 점이 중요하다. 학습은 역전파에 따른 gradient 저장으로 메모리가 병목이고, 추론은 **KV cache 관리**와 **저지연 행렬 곱셈**이 병목이다. 이번 인프라 재편은 Ch04~Ch06에서 다룰 모델 배포 관점에서 실무적 의미가 크다.

### 5. Amazon, Anthropic에 최대 $25B 추가 투자 — AI 인프라 장기 락인 확대
**출처:** [CNBC — 2026-04-20](https://www.cnbc.com/2026/04/20/amazon-invest-up-to-25-billion-in-anthropic-part-of-ai-infrastructure.html)

Amazon이 Anthropic에 즉시 $5B, 향후 커머셜 마일스톤 달성 시 최대 $20B 추가 투자를 약정했다. 총액은 기존 투자와 합쳐 수백억 달러에 이르며, Anthropic은 AWS Trainium 칩 기반의 대규모 학습 인프라를 독점적으로 확보한다. 이 거래는 Anthropic 매출이 2025년 말 $9B → 2026년 3월 말 $30B로 3개월 만에 3배 이상 폭증한 직후 공개됐으며, 일부 VC는 $800B 가치의 선제 투자 제안까지 내놓은 상태다.

> **학습 연결:** 모델 학습은 막대한 TPU/Trainium/GPU 연산을 요구한다. Anthropic-Amazon, OpenAI-Microsoft, xAI-Oracle처럼 **클라우드 인프라 파트너십이 곧 모델 경쟁력**이 되는 구조가 고착화되고 있다. Ch08~Ch09에서 배울 LangChain·RAG를 실전 운영할 때도 어느 클라우드·어느 API를 선택하는가가 비용 구조의 핵심이 된다.
