### 1. Anthropic Mythos — 10조 파라미터 Transformer 모델, 모든 주요 OS·브라우저 보안 취약점 자동 발견
**출처:** [Bloomberg — 2026-04-10](https://www.bloomberg.com/news/articles/2026-04-10/anthropic-model-scare-sparks-urgent-bessent-powell-warning-to-bank-ceos)

Anthropic의 Mythos 모델(10조 파라미터, MoE 아키텍처)이 사이버보안을 별도로 학습하지 않았음에도 모든 주요 OS와 웹 브라우저에서 수천 개의 제로데이 취약점을 자동 발견했다. OpenBSD의 27년 된 버그까지 찾아내고, 완전한 다단계 공격 체인(횡이동→권한 상승→데이터 유출)을 자율 구성하는 능력을 보였다. 미 재무장관 Bessent과 연준 의장 Powell이 Citi·Goldman Sachs 등 월가 CEO를 긴급 소집했으며, 캐나다 중앙은행(4/10)·영란은행(4/11)도 자국 금융기관과 비상 논의에 돌입했다. Anthropic은 Project Glasswing을 통해 AWS·Apple·Google·Microsoft·Nvidia·JPMorgan 등 약 40개 파트너에게만 접근을 허용하고 있으며, 4/12에는 트럼프 행정부가 오히려 은행들에 Mythos 테스트를 권장하는 움직임도 보도되었다.

> **학습 연결:** Transformer의 self-attention mechanism은 입력 시퀀스 내 토큰 간 장거리 의존성을 포착하는 데 특화되어 있다. Mythos는 소스코드를 토큰 시퀀스로 처리하면서 함수 호출 패턴·메모리 접근 순서·조건 분기 구조에서 "알려진 CVE와 구조적으로 유사한 패턴"을 자동 인식하는데, 이는 자연어에서 주어-술어 관계를 파악하는 것과 동일한 attention 원리의 확장이다. Ch04에서 배우는 self-attention의 Query-Key-Value 연산이 자연어를 넘어 형식 언어(코드)까지 처리할 수 있음을 보여주는 사례다.

### 2. 소프트뱅크·NEC·소니·혼다, 'Japan AI Alliance' 설립 — 1조엔 정부 지원으로 1조 파라미터 모델 개발
**출처:** [The Japan Times — 2026-04-12](https://www.japantimes.co.jp/business/2026/04/12/companies/softbank-ai-new-firm/)

소프트뱅크 주도로 NEC·소니·혼다가 각각 10% 이상 지분을 투자해 'ニホンAI基盤モデル開発(니혼AI기반모델개발)' 법인을 설립했다. 일본제철·미쓰비시UFJ·미쓰이스미토모·미즈호 등 일본 주요 기업도 참여해 사실상 산업계 총동원 체제다. 소프트뱅크·NEC가 파운데이션 모델 개발, 소니·혼다가 자동차·로봇·게임·반도체 등 산업 적용을 담당하며, 목표는 1조 파라미터 규모의 피지컬 AI 모델이다. 경산성이 NEDO를 통해 2026 회계연도부터 5년간 총 1조엔(약 9.3조원)을 지원하며, 별도로 Microsoft가 $100억을 일본 AI 인프라에 투자할 계획이다.

> **학습 연결:** 1조 파라미터 모델의 학습에는 대규모 일본어 코퍼스의 토크나이저 설계가 필수다. 일본어는 한국어와 유사하게 교착어적 특성과 한자·히라가나·가타카나 혼용 문제가 있어, Ch04에서 배우는 BPE·SentencePiece 등 서브워드 토크나이저의 언어별 최적화 전략이 모델 성능을 좌우한다. 한국의 소버린 AI(정부 주도 5개 컨소시엄 경쟁)와 일본의 접근법(민간 주도 단일 법인) 비교도 흥미로운 관전 포인트다.

### 3. MiniMax M2.7 오픈소스 공개 — '자가 진화(Self-Evolving)' 학습으로 Claude Opus 능가
**출처:** [MarkTechPost — 2026-04-12](https://www.marktechpost.com/2026/04/12/minimax-just-open-sourced-minimax-m2-7-a-self-evolving-agent-model-that-scores-56-22-on-swe-pro-and-57-0-on-terminal-bench-2/)

중국 AI 기업 MiniMax가 M2.7 모델을 4/11 오픈소스로 공개했다. 230B 파라미터의 Sparse MoE 아키텍처로, 62개 레이어에 256개 로컬 전문가(expert) 중 토큰당 8개만 활성화되어 실제 추론 시 10B 파라미터만 사용한다. 가장 독특한 점은 '자가 진화' 학습 — 모델이 자율적으로 "실패 분석→변경 계획→코드 수정→평가→비교→유지/롤백" 루프를 100회 이상 반복하며 자체 개발 워크플로우의 30~50%를 인간 개입 없이 처리했다. SWE-bench Verified 78%(Claude Opus 55% 대비 23%p 우위), SWE-Pro 56.22%(GPT-5.3-Codex 동등), 속도는 3배 빠르다.

> **학습 연결:** MoE의 "토큰당 8개 전문가 활성화"는 Ch04의 attention mechanism에서 Query-Key 내적으로 관련 토큰에 가중치를 집중하는 것과 구조적으로 유사한 "선택적 연산" 원리다. 전체 256개 전문가 중 입력 토큰의 특성에 따라 라우터(router)가 8개를 선택하는 과정이 attention의 softmax 가중치 배분과 같은 역할을 한다. 이를 통해 230B 모델을 10B 수준의 연산 비용으로 추론할 수 있다.

### 4. Pinterest, 프로덕션 MCP 에코시스템 구축 — AI 에이전트 월 66,000회 자동화, 7,000시간 절감
**출처:** [InfoQ — 2026-04-10](https://www.infoq.com/news/2026/04/pinterest-mcp-ecosystem/)

Pinterest 엔지니어링 팀이 MCP(Model Context Protocol)를 프로덕션 규모로 배치한 사례를 공개했다. 핵심은 하나의 거대한 서버가 아닌 Presto·Spark·Airflow 등 도메인별 전용 MCP 서버 아키텍처로, 컨텍스트 비대화를 방지하고 세분화된 접근 제어를 구현했다. MCP 레지스트리가 중앙 소스 오브 트루스 역할을 하며, 모든 서버는 보안·법무/프라이버시·GenAI 리뷰를 통과해야 프로덕션 배치 가능하다. 성과는 월 66,000회 호출, 월 7,000시간 절감 — "질문에 답하는" 수준이 아니라 "엔지니어링 태스크를 자동화하는" 에이전트로 MCP를 활용한다.

> **학습 연결:** MCP 기반 AI 에이전트가 Presto 쿼리를 작성하거나 Airflow DAG를 수정하려면, 자연어 명령을 코드로 변환하는 NLU(Natural Language Understanding) 파이프라인이 필수다. Ch04에서 배우는 시퀀스-투-시퀀스 모델의 인코더-디코더 구조가 "자연어 입력 → 구조화된 명령 출력" 변환의 기반 아키텍처이며, attention mechanism이 입력 문장의 어떤 부분이 출력의 어떤 부분에 대응하는지를 결정한다.

### 5. Microsoft Agent Framework 1.0 정식 출시 — MCP·A2A 프로토콜 통합, 에이전트 프레임워크 경쟁 본격화
**출처:** [Microsoft DevBlogs — 2026-04-03](https://devblogs.microsoft.com/agent-framework/microsoft-agent-framework-version-1-0/)

Microsoft가 Semantic Kernel + AutoGen을 통합한 Agent Framework 1.0을 정식 출시했다. MCP 지원(에이전트의 외부 도구 동적 발견·호출), A2A(Agent-to-Agent) 프로토콜(다른 프레임워크 에이전트와 크로스 런타임 협업), DevUI(브라우저 기반 에이전트 실행·메시지 흐름·도구 호출 실시간 디버거) 등 세 가지 핵심 기능을 갖췄다. .NET과 Python 양쪽 SDK를 지원하며 LTS를 약속했다. Anthropic Agent SDK, Google ADK, LangGraph와 함께 에이전트 프레임워크 경쟁 구도가 명확해졌다.

> **학습 연결:** Agent Framework의 MCP 도구 호출 과정에서 모델은 자연어 요청을 분석해 적절한 도구를 선택하고 파라미터를 생성하는데, 이는 Ch04에서 배우는 Transformer의 디코딩 과정(다음 토큰 예측)이 확장된 형태다. "function calling"은 디코더가 일반 텍스트 대신 구조화된 JSON을 생성하도록 학습된 것으로, 어휘 사전(vocabulary)에 함수명과 파라미터가 포함된 특수 토큰이 추가된 것이라 볼 수 있다.
