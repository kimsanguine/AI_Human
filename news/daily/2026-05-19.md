## TODAY'S TOP AI

### 1. 📰 Thinking Machines Lab, "듣는 동시에 말하는" Interaction Model 공개 — 턴테이킹 0.40초, GPT-Realtime의 1/3
**출처:** [VentureBeat — 2026-05-11](https://venturebeat.com/technology/thinking-machines-shows-off-preview-of-near-realtime-ai-voice-and-video-conversation-with-new-interaction-models)

Mira Murati(전 OpenAI CTO)의 Thinking Machines Lab이 **TML-Interaction-Small**을 공개했다. 276B MoE(활성 12B) 구조로 STT→LLM→TTS의 3단 파이프라인이 아닌 **단일 모델 안에서 듣기·말하기·시각 처리를 동시 진행**한다. 턴테이킹 지연 시간이 **0.40초** — Gemini-3.1-flash-live(0.57s)·GPT-realtime-2.0(1.18s)보다 2~3배 빠르며 인간 대화 속도에 근접했다. 사용자가 말하는 도중 "응응", "오 그래?" 같은 백채널 추임새를 자연스럽게 흘려보낼 수 있다.

> **학습 연결:** Ch05의 기존 STT+TTS 분리 파이프라인을 **End-to-End Speech-to-Speech 모델**이 어떻게 대체하는지 보여주는 결정적 사례. Whisper(STT) → LLM → ElevenLabs(TTS)로 이어지는 3단 구조의 누적 지연(800ms+)을 단일 모델 1회 추론(400ms)으로 압축하는 아키텍처적 전환을 체감할 수 있다.

### 2. 📰 Google, Gboard에 Gemini 받아쓰기 "Rambler" 탑재 — Wispr Flow·Typeless 시장 정조준
**출처:** [TechCrunch — 2026-05-12](https://techcrunch.com/2026/05/12/google-adds-gemini-powered-dictation-to-gboard-which-could-be-bad-news-for-dictation-startups/)

Google이 Android Show: I/O Edition에서 Gboard 기본 키보드에 **Gemini 기반 받아쓰기 Rambler**를 공식 도입했다. 핵심은 세 가지: (1) "어, 음" 같은 필러 자동 제거, (2) 문장 중간 수정 인식("3시…아니, 2시"), (3) **코드 스위칭** — 영어로 말하다 한국어로 바꿔도 문맥 유지. Galaxy·Pixel 우선 출시, 음성 녹음은 저장하지 않고 전사용으로만 처리. 직격탄을 맞는 곳은 Wispr Flow($2B 밸류, 5/13 다룬 곳)·Typeless 같은 STT 받아쓰기 스타트업이다.

> **학습 연결:** Ch05에서 배우는 STT 모델이 **OS 기본 키보드 레이어로 내려간 순간** — 더 이상 별도 앱이 아니라 "타이핑 = 말하기"가 디폴트가 된다. Whisper 같은 범용 STT와 Gemini Multilingual처럼 코드스위칭을 지원하는 멀티링구얼 STT의 차이가 왜 사업적으로 결정적인지 체감 가능.

### 3. 📰 Anthropic, OpenAI·Google이 쓰던 SDK 자동 생성 스타트업 Stainless 인수
**출처:** [TechCrunch — 2026-05-18](https://techcrunch.com/2026/05/18/anthropic-has-acquired-the-dev-tools-startup-used-by-openai-google-and-cloudflare/)

Anthropic이 SDK 자동 생성 스타트업 **Stainless**(Sequoia·a16z 투자, 인수가 $300M+ 추정)를 인수하고 **호스팅 SDK 서비스를 종료**한다고 밝혔다. Stainless는 OpenAI·Google·Cloudflare가 OpenAPI 스펙에서 Python/TypeScript SDK를 자동 생성하는 데 사용해온 핵심 인프라다. 기존 고객의 SDK 사용권은 유지되지만 신규 호스팅은 차단 — 경쟁 AI 랩의 개발자 경험을 직접 통제하려는 수직 통합 신호.

> **학습 연결:** Ch08 Langchain·MCP 시대에 **API 호출 = SDK 의존성**이라는 점을 상기시키는 사건. 우리가 `pip install openai`로 받는 SDK 뒤에 누가 코드를 자동 생성·유지보수하는지(파이프라인의 가시성)를 의식하지 않으면, AI 에이전트 빌드 시 단일 공급자에 종속될 수 있다.

### 4. 💻 k2-fsa/OmniVoice — 3초 샘플로 600+ 언어 제로샷 복제, WER 0.84% (Apache 2.0)
**출처:** [AI타임스 — 2026년 5월 상순](https://www.aitimes.com/news/articleView.html?idxno=209553)

샤오미의 차세대 Kaldi 팀(k2-fsa)이 오픈소스로 푼 **OmniVoice**가 한국 커뮤니티에서도 입소문을 타고 있다. 핵심 차별화는 (1) **확산 언어 모델(Diffusion LM) 기반 비자기회귀 아키텍처** — 텍스트→의미→음향의 2단 파이프라인 대신 텍스트를 곧바로 음향 토큰으로 변환해 **실시간보다 40배 빠른 생성**, (2) **646개 언어** 지원 (Whisper의 99개, ElevenLabs의 36개를 압도), (3) **3~10초 참조 음성으로 제로샷 보이스 클로닝** + 텍스트로 음색 디자인("영국 억양의 낮은 목소리"). 중국어 Seed-TTS 테스트셋 WER **0.84%**로 ElevenLabs·MiniMax를 상회.

> **학습 연결:** Ch05의 핵심 개념인 **Mel-spectrogram·Vocoder 2단 구조 vs. 단일 모델 직접 합성**의 차이를 코드로 만져볼 수 있는 대표 레포. 어제 다룬 VoxCPM2(토크나이저 프리)와 함께 "TTS의 토큰화 패러다임"이 어떻게 깨지고 있는지 비교 학습 추천.

### 5. 📰 멀티 에이전트가 텍스트 대신 임베딩으로 대화한다 — UIUC·Stanford의 RecursiveMAS
**출처:** [VentureBeat — 2026-05-15](https://venturebeat.com/orchestration/how-recursivemas-speeds-up-multi-agent-inference-by-2-4x-and-reduces-token-usage-by-75)

UIUC·Stanford 연구팀이 공개한 **RecursiveMAS**는 에이전트끼리 자연어 대신 **임베딩 벡터**로 정보를 주고받는다. 결과: 코드 생성·의료 추론·검색 도메인에서 **추론 속도 2.4배·토큰 사용량 75% 절감**, 정확도는 오히려 개선. "사람이 읽을 자연어로 메시지를 굳이 디코딩→재인코딩하는 비용"이 멀티 에이전트 시스템의 숨은 병목이었다는 가설이 검증된 셈.

> **학습 연결:** Ch04(임베딩·Attention)와 Ch08(에이전트·LangChain)을 잇는 다리. 우리가 배운 `text-embedding-3` 같은 벡터가 **사람이 읽는 검색 인덱스만이 아니라 에이전트 간 통신 프로토콜**로 쓰일 수 있다는 점 — Ch09 RAG·Ch08 MCP에서 응용 가능한 발상.
