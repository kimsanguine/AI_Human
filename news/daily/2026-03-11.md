### 1. Google DeepMind, Gemini 3.1 Pro 공개 — 100만 토큰 컨텍스트·ARC-AGI-2 77.1% 달성
**출처:** [Crescendo AI / MIT Technology Review](https://www.crescendo.ai/news/latest-ai-news-and-updates)

Google DeepMind의 최신 Gemini 3.1 Pro가 공식 공개됐다. 100만 토큰(약 책 500권 분량)의 컨텍스트 처리가 가능하며, AGI 수준을 측정하는 ARC-AGI-2 벤치마크에서 77.1%를 기록해 현존 최고 수준에 올랐다. 텍스트·이미지·오디오·영상·코드를 아우르는 멀티모달 추론이 특징이다.

> **학습 연결:** Python에서 `google-generativeai` 라이브러리를 사용하면 단 몇 줄의 코드로 Gemini API를 호출할 수 있다. Ch01에서 배우는 Python 기초(변수, 함수, 라이브러리 import)가 이런 최신 AI 모델을 직접 사용하는 첫 번째 단계가 된다.

### 2. OpenAI, GPT-5.4 Thinking 모델 출시 — '생각하는 과정'을 실시간으로 보여준다
**출처:** [TechCrunch](https://techcrunch.com/2026/03/05/openai-launches-gpt-5-4-with-pro-and-thinking-versions/) / [OpenAI](https://openai.com/index/introducing-gpt-5-4/)

OpenAI가 3월 5일 GPT-5.4 Thinking 모델을 출시했다. 기존 Thinking 모델과의 가장 큰 차이는 '응답 중 사고 계획을 미리 보여주는 기능'으로, 사용자가 AI가 답변하는 도중에도 방향을 수정할 수 있다. 동일 문제를 이전 모델보다 훨씬 적은 토큰으로 해결하며, 코딩·컴퓨터 사용·도구 검색에서 최고 수준의 성능을 기록했다. Plus·Team·Pro 사용자에게 우선 제공된다.

> **학습 연결:** GPT-5.4 Thinking이 코딩에 강한 이유 중 하나가 Python의 명확한 문법이다. 타입 힌팅과 독스트링처럼 '의도를 코드로 표현하는 법'을 Ch01부터 배우면, AI 코딩 도구와의 협업 품질이 처음부터 달라진다.

### 3. 2026년 AI 에이전트, '도구'에서 '디지털 동료'로 진화 — IBM·Microsoft 공동 전망
**출처:** [IBM Think / Microsoft News](https://news.microsoft.com/source/features/ai/whats-next-in-ai-7-trends-to-watch-in-2026/)

IBM과 Microsoft가 발표한 2026 AI 트렌드 보고서에 따르면, AI 에이전트는 올해를 기점으로 개인 생산성 도구를 넘어 팀 단위 워크플로우 오케스트레이션 플랫폼으로 확장된다. 단순히 질문에 답하는 것에서 나아가 '필요를 예측하고 능동적으로 협업'하는 방향으로 진화하는 중이며, 에이전트 생성의 민주화도 빠르게 진행되고 있다.

> **학습 연결:** AI 에이전트를 만들고 제어하는 핵심 언어가 바로 Python이다. Ch08(LangChain)과 Ch09(RAG)에서 직접 에이전트를 구현하게 되는데, 그 기초가 지금 배우는 Python 문법이다.

### 4. Anthropic, Claude Code에 'Code Review' 기능 출시 — AI가 만든 코드를 AI가 검토한다
**출처:** [TechCrunch](https://techcrunch.com/2026/03/09/anthropic-launches-code-review-tool-to-check-flood-of-ai-generated-code/) / [The New Stack](https://thenewstack.io/anthropic-launches-a-multi-agent-code-review-tool-for-claude-code/)

Anthropic이 3월 9일 Claude Code에 멀티 에이전트 기반 코드 리뷰 기능을 출시했다. GitHub Pull Request에 자동으로 연동되어 여러 AI 에이전트가 병렬로 로직 오류·버그·보안 취약점을 분석하고 코드에 직접 코멘트를 남긴다. PR 크기에 따라 건당 $15~25 수준이며 평균 20분 소요. 'Vibe Coding'으로 급증한 AI 생성 코드의 품질 문제를 해결하기 위한 도구로, Uber·Salesforce·Accenture 등 엔터프라이즈 고객 대상으로 리서치 프리뷰 중이다.

> **학습 연결:** AI가 코드를 생성하고 AI가 리뷰하는 시대가 왔다. 하지만 어떤 버그가 왜 문제인지 판단하는 것은 여전히 개발자의 몫이다. Python 기초 문법(조건문, 반복문, 함수)을 제대로 배워야 AI 리뷰 코멘트를 해석하고 수정할 수 있다.

### 5. Astral, Rust 기반 초고속 Python 타입 체커 `ty` 발표 — uv·Ruff 팀의 새 도구
**출처:** [DasRoot.net — Python Libraries Shaping AI in 2026](https://dasroot.net/posts/2026/03/python-libraries-shaping-ai-2026/)

`uv` 패키지 매니저와 `Ruff` 린터를 만든 Astral 팀이 새로운 Python 타입 체커 `ty`를 발표했다. Rust로 작성되어 기존 타입 체커(mypy, pyright) 대비 압도적인 속도를 자랑하며, AI 코딩 어시스턴트가 타입 힌트를 기반으로 더 정확한 코드 제안을 할 수 있도록 지원한다. 같은 생태계에서 `complexipy`(코드 복잡도 분석)와 `Kreuzberg`(문서 → LLM 변환) 도 함께 주목받고 있다.

> **학습 연결:** Python 타입 힌팅(`def greet(name: str) -> str:`)은 수업 후반부에 등장하는 중요한 개념이다. 타입 힌팅을 잘 쓸수록 AI 코딩 도구(Copilot, Cursor)가 더 정확한 코드를 제안해 준다. 현대 Python 개발의 필수 습관이다.
