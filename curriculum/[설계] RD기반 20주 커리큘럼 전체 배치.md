### Q. 핵심 발견 — 이전 설계와 무엇이 다른가?

**발견 1: 프로젝트가 전체의 35% (280h/7주)**

이전 설계에서 "Phase 5 캡스톤 3주"로 프로젝트를 배치했지만, 실제로는 7주(35일) 분량. 3주로는 절반도 못 담는다.

**발견 2: 프로젝트는 과목과 인터리빙**

RD의 3개 프로젝트는 각각 관련 과목 직후에 배치되는 구조:
- PJ1(음성모델) → 과목4+5 직후
- PJ2(LLM PERSO AI) → 과목6+7 직후
- PJ3(RAG PERSO AI) → 과목8+9 직후

**발견 3: Python은 2주면 충분**

이전 설계의 "Phase 1 = W1-5 (5주 Python)"은 RD 기준 80h(2주)와 큰 괴리. Python에 5주를 쓸 근거가 RD에 없다.

**발견 4: 과목4 NLP+음성이 96h로 최대 과목**

NLP+음성 데이터가 12일(96h)로 가장 비중이 크고, 과목5 TTS/STT(64h/8일)와 합치면 20일(4주). 음성AI 파트가 과정의 핵심.

## 2. 수정된 5Phase 구조

### Q. 왜 Phase 구조를 수정해야 하는가?

| 구분 | 이전 설계 | RD 기반 수정안 | 차이 |
|------|---------|-------------|------|
| Phase 1 | W1-5 (5주) Python | **W1-2 (2주)** Python | -3주 |
| Phase 2 | W6-10 (5주) ML/DL/NLP | **W3-6 (4주)** ML/DL + NLP | -1주, 위치 변경 |
| Phase 3 | W11-13 (3주) LLM+PE+음성 | **W7-10 (4주)** 음성AI + PJ1 | +1주, 내용 변경 |
| Phase 4 | W14-17 (4주) LC+RAG+Agent | **W11-15 (5주)** LLM+PE+LoRA+PJ2 | +1주, 내용 변경 |
| Phase 5 | W18-20 (3주) 캡스톤 | **W16-20 (5주)** LC+RAG+Agent+PJ3 | +2주, 내용 변경 |

### Q. 수정된 Phase 구조의 핵심 논리는?

```
Phase 1 (W1-2)   : Python 기초          → "코딩 체력"
Phase 2 (W3-6)   : ML/DL + NLP          → "AI 기초 체력"
Phase 3 (W7-10)  : 음성AI + PJ1         → "첫 번째 전문성 + 첫 프로젝트"
Phase 4 (W11-15) : LLM + PE + LoRA + PJ2 → "LLM 마스터 + 두 번째 프로젝트"
Phase 5 (W16-20) : LC + RAG + Agent + PJ3 → "실전 시스템 + 최종 프로젝트"
```

**설계 원칙: "과목 → 프로젝트" 사이클**

각 Phase 후반에 PERSO AI 프로젝트가 배치되어, 배운 것을 바로 프로젝트로 연결:
- Phase 3 후반: PJ1 음성모델
- Phase 4 후반: PJ2 LLM PERSO AI
- Phase 5 후반: PJ3 RAG PERSO AI
